# 架构说明

## 1. 设计边界

`grid-agent` 的第一版只解决一个可验证问题：给定树状 10 kV 馈线、节点电压、支路零序电流以及保护/开关动作，输出最可能的故障区段和证据链。

当前版本不把规则评分称为暂态解析解，也不直接执行遥控操作。

## 2. 核心分层

```text
JSON/SCADA/录波适配器
        ↓
FeederCase 领域模型
        ↓
RadialFeeder 拓扑约束
        ↓
FaultDiagnoser 证据评分
        ↓
诊断结果 / 报告 / Agent 工具调用
```

### model

保存节点、支路、量测和案例，不掺入算法逻辑。

### topology

验证网络是否满足：

- 单一源节点；
- 每个非源节点只有一条入边；
- 无环；
- 所有节点均可由源节点到达。

同时提供唯一供电路径、上下游节点和支路深度查询。

### diagnosis

当前评分由四项证据组成：

| 证据 | 默认权重 |
|---|---:|
| 支路零序电流 | 0.45 |
| 下游电压跌落 | 0.25 |
| 上下游拓扑边界反差 | 0.20 |
| 保护或开关动作 | 0.10 |

这些权重是工程启动值，后续必须通过现场样本和电磁暂态仿真标定。

## 3. 后续模块

### transient

面向连云港树状配电网暂态解析研究：

```text
故障源项
× 树上唯一路径传递函数
× 分支反射/透射
× 末端边界反射
× 传播衰减
```

输出优先采用可用于选线、区段定位和故障类型判断的特征，而不是盲目重建完整波形。

### adapters

计划分别实现：

- PSCAD 工程与批处理适配；
- MATLAB Engine/脚本适配；
- PSS®E Python API 适配；
- CDEGS 数据导入导出；
- SCADA、保护动作、录波和拓扑数据适配。

### agent

Agent 只负责流程编排、工具选择、证据汇总和报告生成；数值计算继续交给经过验证的专业软件和确定性算法。

## 4. 对接既有项目

- **公司 M4**：拓扑图模型、准静态状态估计残差、小电流接地选线、FA 隔离与恢复。
- **连云港**：树状配电网故障暂态响应解析解与工程应用。
- **河北中调课题三**：运行风险识别、场景推演和调控建议，后续通过独立场景模型接入。

## 5. 工程原则

1. 过程成果不能冒充定稿。
2. 没有数据或依据时明确缺失，不编造结论。
3. 每个诊断结论保留输入、算法版本和证据链。
4. Agent 建议不得绕过保护、调度规程和现场安全确认。
